Project Overview

Cabin Escape is a puzzle-adventure game developed during the Game Developers United (GDU) Game Jam 2024. The theme for the jam was "Cabin," and I took on the challenge of creating an immersive escape room experience set in a mysterious, abandoned cabin in the woods.

As the sole programmer and designer, I leveraged my expertise in Unreal Engine 5 and C++ to create a captivating game in just 48 hours. The game features custom Interaction and Inventory Systems, atmospheric lighting, and challenging puzzles that test players' problem-solving skills.

Project Details

  • Engine: Unreal Engine 5
  • Team Size: 1 (Solo Project)
  • Role: Solo Developer (Programming, Design, Art)
  • Development Time: 48 hours
Play Demo View on GitHub

Key Features

  • Custom Interaction System: Allows players to examine and interact with various objects in the cabin.
  • Inventory Management: Players can collect, combine, and use items to solve puzzles.
  • Dynamic Lighting: Utilizes Unreal Engine 5's Lumen technology for realistic, real-time global illumination.
  • Atmospheric Audio: Immersive sound design enhances the eerie cabin atmosphere.
  • Multiple Endings: Player choices and puzzle-solving speed affect the game's outcome.
  • Optimized Performance: Smooth gameplay even on mid-range hardware.

Interaction System

// InteractionComponent.h
UCLASS(ClassGroup=(Custom), meta=(BlueprintSpawnableComponent))
class CABINESCAPE_API UInteractionComponent : public UActorComponent
{
    GENERATED_BODY()

public:
    UFUNCTION(BlueprintCallable, Category = "Interaction")
    void Interact(AActor* Interactor);

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Interaction")
    FString InteractionPrompt;

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Interaction")
    bool bCanInteract = true;

    DECLARE_DYNAMIC_MULTICAST_DELEGATE_OneParam(FOnInteractionDelegate, AActor*, Interactor);
    UPROPERTY(BlueprintAssignable, Category = "Interaction")
    FOnInteractionDelegate OnInteraction;
};

// InteractionComponent.cpp
void UInteractionComponent::Interact(AActor* Interactor)
{
    if (bCanInteract)
    {
        OnInteraction.Broadcast(Interactor);
    }
}

Inventory System

+ 

// InventoryComponent.h
UCLASS(ClassGroup=(Custom), meta=(BlueprintSpawnableComponent))
class CABINESCAPE_API UInventoryComponent : public UActorComponent
{
    GENERATED_BODY()

public:
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Inventory")
    TArray Items;

    UFUNCTION(BlueprintCallable, Category = "Inventory")
    bool AddItem(UInventoryItem* Item);

    UFUNCTION(BlueprintCallable, Category = "Inventory")
    bool RemoveItem(UInventoryItem* Item);

    UFUNCTION(BlueprintCallable, Category = "Inventory")
    UInventoryItem* FindItemByClass(TSubclassOf ItemClass);
};

// InventoryComponent.cpp
bool UInventoryComponent::AddItem(UInventoryItem* Item)
{
    if (Item)
    {
        Items.Add(Item);
        return true;
    }
    return false;
}

bool UInventoryComponent::RemoveItem(UInventoryItem* Item)
{
    return Items.Remove(Item) > 0;
}

UInventoryItem* UInventoryComponent::FindItemByClass(TSubclassOf ItemClass)
{
    for (UInventoryItem* Item : Items)
    {
        if (Item->GetClass() == ItemClass)
        {
            return Item;
        }
    }
    return nullptr;
}

Puzzle Logic

+ 

// SafePuzzle.h
UCLASS(ClassGroup=(Custom), meta=(BlueprintSpawnableComponent))
class CABINESCAPE_API USafePuzzle : public UActorComponent
{
    GENERATED_BODY()

public:
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Puzzle")
    FString CorrectCombination;

    UFUNCTION(BlueprintCallable, Category = "Puzzle")
    bool CheckCombination(const FString& InputCombination);

    DECLARE_DYNAMIC_MULTICAST_DELEGATE(FOnPuzzleSolvedDelegate);
    UPROPERTY(BlueprintAssignable, Category = "Puzzle")
    FOnPuzzleSolvedDelegate OnPuzzleSolved;
};

// SafePuzzle.cpp
bool USafePuzzle::CheckCombination(const FString& InputCombination)
{
    if (InputCombination == CorrectCombination)
    {
        OnPuzzleSolved.Broadcast();
        return true;
    }
    return false;
}

Lessons Learned and Challenges Faced

Lessons Learned:

  • Rapid Prototyping: The importance of quickly implementing core mechanics and iterating on them.
  • Scope Management: Learning to prioritize essential features and cut non-critical elements to meet the tight deadline.
  • Performance Optimization: Balancing visual fidelity with performance, especially when using Unreal Engine 5's advanced features.
  • Asset Creation Pipeline: Developing a streamlined workflow for creating and importing assets efficiently.

Challenges Faced:

  • Time Constraint: Developing a complete game in 48 hours required careful planning and efficient use of time.
  • Feature Creep: Resisting the temptation to add unnecessary features that could jeopardize the project's completion.
  • Performance Issues: Initially, the game suffered from performance problems due to the complexity of the lighting system. This required optimization of the level design and careful use of Lumen settings.
  • Puzzle Balancing: Creating puzzles that were challenging but not frustratingly difficult, especially without time for extensive playtesting.
  • Bug Fixing Under Pressure: Identifying and fixing critical bugs quickly while maintaining the development momentum.
